What Gets Deployed — k8s/
namespaces/ Namespace add-env
One YAML per environment (dev.yaml, staging.yaml, prod.yaml). Defines the Kubernetes namespace resource.
apps/<app>/base/ Kustomization add-app
Kustomize base shared across all environments. References deployment.yaml and service.yaml.
apps/<app>/overlays/<env>/ Kustomization add-env add-app
Environment-specific Kustomize overlay. Sets image tags, replicas, and env vars. References ../base.
How It Gets Deployed — argocd/
parent-app.yaml Application init
App-of-apps root Application. Watches argocd/apps/ and auto-syncs all Application manifests found there.
apps/projects.yaml Application init
Dedicated Application that watches argocd/projects/ and syncs AppProject CRDs.
apps/<app>-<env>.yaml Application add-env add-app
One ArgoCD Application per app+env combo. Points to the matching Kustomize overlay. Deploys to the env namespace.
projects/<name>.yaml AppProject add-project
AppProject resources for RBAC. Controls which repos and namespaces apps can access.
Generators
infra-ctl.sh
Manages repo structure. Commands: init, add-app, add-env, add-project, edit-project.
cluster-ctl.sh
Manages local k3d cluster. Commands: init-cluster, delete-cluster, status.
templates/
{{PLACEHOLDER}} templates rendered by bash parameter expansion. Mirrors the output directory structure.

Relationships
parent-app.yaml → argocd/apps/* watches directory, auto-syncs all Applications
apps/projects.yaml → argocd/projects/* watches directory, syncs AppProject CRDs
<app>-<env>.yaml → k8s/apps/<app>/overlays/<env>/ source path for Kustomize build
overlays/<env>/kustomization.yaml → ../../base/ inherits and patches the base resources
<app>-<env>.yaml → projects/<name>.yaml spec.project references an AppProject for RBAC
