What Gets Deployed — k8s/
namespaces/ Namespace add-env
One YAML per environment (dev.yaml, staging.yaml). Defines the Kubernetes namespace.
apps/<app>/base/ Kustomization add-app
Kustomize base shared across environments. References deployment.yaml and service.yaml.
apps/<app>/overlays/<env>/ Kustomization add-env add-app
Environment overlay. Sets image tags, replicas, env vars. Sealed secrets live here too.
platform/ RBAC user-ctl
ClusterRoles, Roles, and Bindings generated by user-ctl.sh add-role.
How It Gets Deployed — argocd/
parent-app.yaml Application init
App-of-apps root. Watches argocd/apps/ and syncs all Application manifests.
apps/<app>-<env>.yaml Application add-env add-app
One Application per app+env. Points to the Kustomize overlay, deploys to env namespace.
projects/<name>.yaml AppProject add-project
AppProject for RBAC. Controls which repos and namespaces apps can access.
Progressive Delivery — kargo/ optional
promotion-order.txt init
Linear pipeline definition. One env per line (default: dev, staging, production).
<app>/warehouse.yaml Warehouse add-app
Watches container registry for new image tags. One per app.
<app>/<env>-stage.yaml Stage add-env add-app
Promotion stage. Updates overlay image tag, commits, triggers ArgoCD sync.
Secrets — sealed secrets
.sealed-secrets-cert.pem secret-ctl init
Public cert for encrypting secrets. Committed to repo, safe to share.
overlays/<env>/sealed-secret.yaml SealedSecret secret-ctl add
Encrypted secret per app/env. Decrypted in-cluster by the Sealed Secrets controller.
.sealed-secrets-key.json gitignored
Private key backup. Never committed. Required for disaster recovery only.

Config — .infra-ctl.conf
REPO_URL — Git repo for ArgoCD Application manifests
REPO_OWNER — GitHub owner for image defaults
KARGO_ENABLED — Enable Kargo resource generation (optional)
Helm Values — helm/argocd-values.yaml
configs.cm — ArgoCD accounts (managed by user-ctl)
configs.rbac.policy.csv — Casbin policy (managed by user-ctl)
Relationships
parent-app.yaml → argocd/apps/* watches & syncs Applications
<app>-<env>.yaml → overlays/<env>/ Kustomize source path
Warehouse → ghcr.io registry watches for new image tags
Stage → overlays/<env>/ updates image tag via git commit
Stage → <app>-<env>.yaml triggers ArgoCD sync
SealedSecret → Secret (in-cluster) decrypted by controller
