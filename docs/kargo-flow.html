What each Stage does during promotion
1
Clear git worktree to ensure a clean state before making changes
2
Set image tag in the overlay via kustomize-set-image (updates kustomization.yaml)
3
Commit and push the change to the GitOps repo with a descriptive message
4
Trigger ArgoCD sync on the matching Application (e.g., backend-dev) so changes deploy immediately
kargo/promotion-order.txt defines the pipeline order (dev, staging, production)
kargo/<app>/warehouse.yaml configures which registry + image to watch
